## Block Definition Migration Prompt

Use this prompt to migrate Universal Editor block definition JSON files from the legacy format to the new format.

---

### Important Notes

**⚠️ Skip Columns Blocks:** Do NOT migrate blocks that contain `"behaviour": "columns"` in their definition. These blocks use a special columns behavior and do not require migration.

---

### Migration Rules

Apply the following transformations to each block definition JSON file:

#### 1. Add explicit `model` and `filter` properties to definitions

For **parent/container definitions** (e.g., `accordion`, `carousel`, `cards`):
```json
// ADD these properties at definition level:
"model": "<definition-id>",
"filter": "<definition-id>"
```

For **child/item definitions** (e.g., `accordion-item`, `carousel-item`, `card`):
```json
// ADD this property at definition level:
"model": "<definition-id>"
```

#### 2. Replace `unsafeHTML` with `rows` and `columns`

If the `plugins.da` object contains `unsafeHTML`, replace it:

```json
// BEFORE
"plugins": {
  "da": {
    "unsafeHTML": "<div class=\"block-name\">...</div>"
  }
}

// AFTER
"plugins": {
  "da": {
    "rows": 1,
    "columns": <number-of-columns>
  }
}
```

Determine `columns` based on the block's table structure (typically 1 or 2).

#### 3. Add `fields` array to child definitions

For each child/item definition, add a `fields` array inside `plugins.da` that maps semantic field names to CSS selectors:

```json
// BEFORE (no fields array)
"plugins": {
  "da": {
    "name": "accordion-item",
    "rows": 2,
    "columns": 0
  }
}

// AFTER (with fields array)
"plugins": {
  "da": {
    "name": "accordion-item",
    "rows": 2,
    "columns": 0,
    "fields": [
      { "name": "<semantic-name>", "selector": "<css-selector>" },
      { "name": "<semantic-name>", "selector": "<css-selector>" }
    ]
  }
}
```

#### 4. Convert model field names from selectors to semantic names

In the `models` array, replace CSS selector-based `name` values with semantic names:

```json
// BEFORE
{
  "component": "richtext",
  "name": "div:nth-child(1)",
  "label": "Summary"
}

// AFTER
{
  "component": "richtext",
  "name": "summary",
  "label": "Summary"
}
```

**Naming convention:** Use camelCase, derived from the `label` value:
- "Summary" -> `summary`
- "Background Image" -> `backgroundImage`
- "Image Alt" -> `imageAlt`

#### 5. Simplify CSS selectors in the `fields` array

When creating selectors for the `fields` array, simplify redundant `:nth-child()` specifiers:

```
// BEFORE (overly specific)
div:nth-child(1)>picture:nth-child(1)>img:nth-child(3)[src]

// AFTER (simplified)
div:nth-child(1)>picture>img[src]
```

Keep `:nth-child()` only where needed to distinguish between siblings.

---

### Special Case: Key/Value Blocks

**Identifying Key/Value Blocks:**

Key/value blocks can be identified by the following characteristics:
- Contains `unsafeHTML` with a **two-column HTML structure**
- The first column contains key names (e.g., `<div>recid</div>`, `<div>currentSku</div>`)
- The second column contains value placeholders (e.g., `<div><p>PLACEHOLDER</p></div>`)
- Model field names (CSS selectors) only address the **second column** cells: `div:nth-child(1)>div:nth-child(2)>p:nth-child(1)`

**Example OLD format:**
```json
{
  "definitions": [{
    "plugins": {
      "da": {
        "unsafeHTML": "<div class=\"product-recommendations\"><div><div>recid</div><div><p>PLACEHOLDER</p></div></div><div><div>currentSku</div><div><p>PLACEHOLDER</p></div></div></div>"
      }
    }
  }],
  "models": [{
    "fields": [
      {
        "component": "text",
        "name": "div:nth-child(1)>div:nth-child(2)>p:nth-child(1)",
        "label": "Rec ID"
      }
    ]
  }]
}
```

**Migration Steps for Key/Value Blocks:**

1. **Extract field names from the first column** of the `unsafeHTML` structure
   - Look at each `<div><div>KEY_NAME</div><div>...</div></div>` pair
   - Extract KEY_NAME values (e.g., "recid", "currentSku")
   - Convert to camelCase if needed (e.g., "recid" → "recId")

2. **Remove the entire `unsafeHTML` property**

3. **Replace with simplified DA plugin:**
   ```json
   "plugins": {
     "da": {
       "type": "key-value-block"
     }
   }
   ```

4. **Update model field names** to use the extracted key names from step 1

5. **Add the `model` property** to the definition (same as regular blocks)

**Example NEW format:**
```json
{
  "definitions": [{
    "model": "product-recommendations",
    "plugins": {
      "da": {
        "type": "key-value-block"
      }
    }
  }],
  "models": [{
    "fields": [
      {
        "component": "text",
        "name": "recId",
        "label": "Rec ID"
      },
      {
        "component": "text",
        "name": "currentSku",
        "label": "Current SKU"
      }
    ]
  }]
}
```

---

### Migration Checklist

**Pre-migration:**
- [ ] Check if block has `"behaviour": "columns"` → **Skip migration if present**
- [ ] Identify if block is a **key/value block** (two-column structure with keys in first column)

**For regular blocks:**
- [ ] Add `model` property to all definitions
- [ ] Add `filter` property to parent/container definitions
- [ ] Replace `unsafeHTML` with `rows` and `columns` (if applicable)
- [ ] Add `fields` array to child/item `plugins.da` sections
- [ ] Rename model field `name` values from CSS selectors to semantic names
- [ ] Ensure `fields` array selectors match the original CSS selectors (simplified)
- [ ] Verify field names in `models` match field names in `plugins.da.fields`

**For key/value blocks:**
- [ ] Extract key names from first column of `unsafeHTML`
- [ ] Remove `unsafeHTML` entirely
- [ ] Replace with `"type": "key-value-block"` in DA plugin
- [ ] Update model field names to extracted key names (camelCase)
- [ ] Add `model` property to definition

### Example: Full Migration

**BEFORE:**
```json
{
  "definitions": [
    {
      "title": "Cards",
      "id": "cards",
      "plugins": {
        "da": {
          "unsafeHTML": "<div class=\"cards\"><div><div><picture>...</picture></div><div></div></div></div>"
        }
      }
    },
    {
      "title": "Card",
      "id": "card",
      "plugins": {
        "da": {
          "unsafeHTML": "<div><div><picture>...</picture></div><div></div></div>"
        }
      }
    }
  ],
  "models": [
    {
      "id": "card",
      "fields": [
        {
          "component": "reference",
          "name": "div:nth-child(1)>picture:nth-child(1)>img:nth-child(3)[src]",
          "label": "Image"
        },
        {
          "component": "text",
          "name": "div:nth-child(1)>picture:nth-child(1)>img:nth-child(3)[alt]",
          "label": "Image Alt"
        },
        {
          "component": "richtext",
          "name": "div:nth-child(2)",
          "label": "Text"
        }
      ]
    }
  ],
  "filters": [...]
}
```

**AFTER:**
```json
{
  "definitions": [
    {
      "title": "Cards",
      "id": "cards",
      "model": "cards",
      "filter": "cards",
      "plugins": {
        "da": {
          "rows": 1,
          "columns": 2
        }
      }
    },
    {
      "title": "Card",
      "id": "card",
      "model": "card",
      "plugins": {
        "da": {
          "rows": 1,
          "columns": 2,
          "fields": [
            { "name": "image", "selector": "div:nth-child(1)>picture>img[src]" },
            { "name": "imageAlt", "selector": "div:nth-child(1)>picture>img[alt]" },
            { "name": "text", "selector": "div:nth-child(2)" }
          ]
        }
      }
    }
  ],
  "models": [
    {
      "id": "card",
      "fields": [
        {
          "component": "reference",
          "name": "image",
          "label": "Image"

        },

        }
      ]
    }
  ],
  "filters": [...]
}
```
